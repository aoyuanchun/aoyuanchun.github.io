框架和库的区别

  • 框架：是一套完整的解决方案；对项目的侵入性较大，项目如果需要更换框架，则需要重新架构整个项目。
  • 库（插件）：提供某一个小功能，对项目的侵入性较小，如果某个库无法完成某些需求，可以很容易切换到其它库实现需求。

2.2.0+ 的版本里，当在组件中使用 v-for 时，key 现在是必须的。

  • 当 Vue.js 用 v-for 正在更新已渲染过的元素列表时，它默认用 “就地复用” 策略。如果数据项的顺序被改变，Vue将不是移动 DOM 元素来匹配数据项的顺序， 而是简单复用此处每个元素，并且确保它在特定索引下显示已被渲染过的每个元素。

  • 为了给 Vue 一个提示，以便它能跟踪每个节点的身份，从而重用和重新排序现有元素，你需要为每项提供一个唯一 key 属性。

常见的web服务器软件

Apache HTTP Server

安装步骤：

1、软件解压到c盘下，记得英文命名（不能汉字，一般写wamp）

2、搜索命令右键打开命令（以管理员身份进行 命令行），输入cd （输入apache里bin的路径）回车再输入httpd.exe -k install+回车，然后验证有没有错，httpd.exe -n 然后会报错说C：/Develop/apache/conf/httpd.conf:

所以找到37行的ServerRoot”路径改成apache的根目录” 第二次报错是246行，把DocumentRoot “路径改为apache/htdocs的路径” （DocumentRoot 之后）的 Directory 配置的与 DocumentRoot 路径相同 然后再写一个httpd.exe -n然后会显示成功，就去谷歌搜索127.0.0.1或者输入localhost;如果显示it works；就说明apache下载成功；

IP 地址

Internet Protocol Address

设备在某一个网络中的地址，目前最常见的格式：[0-255].[0-255].[0-255].[0-255] 即为四个 0-255 的数字组成。

作用就是标识一个网络设备（计算机、手机、电视）在某一个具体的网络当中的地址。

127.0.0.1 是本地回环地址（特殊的ip地址，表示本机）

DNS

通过宽带运营商提供的服务器解析一个域名背后的ip地址，这个过程叫DNS寻址，帮你完成DNS寻址过程的服务器叫DNS服务器；

C:\Windows\System32\drivers\etc\hosts 文件是本机的 DNS 依据

注意：

  • 只能影响本机的 DNS 寻址
  • 必须以管理员权限运行的编辑器才有权利修改 hosts 文件

设置本地的hosts文件，与apache是否开启没有本质的联系。如果设置了hosts但是没有开启apache，会显示拒绝连接请求。

请求响应流程

  1. 用户打开浏览器
  2. 地址栏输入我们需要访问的网站网址（URL）
  3. 浏览器会先去缓存里找，没有再去DNS找
  4. 浏览器通过 DNS 服务器获取即将访问的网站 IP
  5. 如果Dns没有就会去网页里找；
  6. 浏览器发起一个对这个 IP 的请求
  7. 服务端接收到这个请求，进行相应的处理（如果是静态文件请求，就读取这个文件）
  8. 服务端将处理完的结果返回给客户端浏览器
  9. 浏览器将服务端返回的结果呈现到界面上

虚拟主机

如果想要在一台机器上部署多个站点，就必须通过配置虚拟主机的方式解决。

1、C:\wamp\Apache24\htdocs去这个地址写访问的网站，然后具体的操作方式就是去C:\wamp\Apache24\conf的httpd.conf里面 httpd.conf 的 505 行注释取消：然后再去apache/conf/extra/去最下面把复制几个，然后把里面的DocumentRoot改成我们创建的网站；”C:\wamp\Apache24\htdocs\taoba”

ServerName www.taobaod.cn

把这两个改成我们自己创建的路径；（访问之前得创建好这个例子，路径是c/wamp/apache/htdocs）
